# 01-PJT 금융 상품 데이터 수집 및 가공 프로젝트

## 프로젝트 개요

본 프로젝트의 필수 과제인 OpenWeatherMap API를 활용하여
**특정 지역의 날씨 데이터를 수집하고,
API 응답 구조를 분석한 뒤 필요한 정보만 가공하여 출력하는 관통 pjt**이다.

필수 과제인 날씨 데이터 수집을 중심으로 구현하였으며,
추가적으로 공공 API 구조 이해를 위해 
금융 상품 데이터 분석을 선택 과제로 진행하였다.

단, 날씨 데이터는 오전 온라인 교육으로 어느정도 수강한 내용이기에
팀원 2명이 각자의 방법으로 수행한 후 방법 비교 및 피드백 진행

## 프로젝트 과정

### 문제 1
**목표**
- key 값 출력: 날씨 데이터의 응답을 JSON 형태로 변환 후 Key 값만 출력하도록 구성
**풀이**
- JSON 응답의 복잡한 중첩 구조를 pprint로 시각화하여 데이터의 흐름을 파악하였다.

### 문제 2
**목표**
- 원하는 값 출력: main, weather 키 값을 딕셔너리로 추출
**풀이**
- 딕셔너리 순회 시 변수에 할당되는 값이 '값(Value)'이 아닌 '키(Key)'라는 점을 명확히 인지하고 조건문을 구성해야 했다.

### 문제 3
**목표**
- Key 한글화: 추출한 키 값을 한글 키로 변환한 새로운 딕셔너리 생성
**풀이**
- API 응답 중 특정 값은 딕셔너리가 아닌 **리스트([])**에 담겨 있다는 것을 확인했다.
- type() 검사를 통해 리스트인 경우 인덱싱([0])을 수행하는 예외 처리 로직을 구현하였다.

### 문제 4
**목표**
- 섭씨 온도 추가: 앞의 결과 딕셔너리에 온도 관련 값을 이용해 섭씨 데이터 필드 추가
**풀이**
- 파이썬의 부동소수점 오차로 인해 부정확한 값이 계산되는 문제가 발생했다.
- Decimal 모듈을 도입하여 정밀한 연산을 수행하고, 최종 출력 시에는 다시 float로 형변환하여 범용성을 확보했다.

### 문제 5
**목표**
- 생성형 AI 활용: OpenWeatherMap API에 대한 프롬프트 구성 및 실행

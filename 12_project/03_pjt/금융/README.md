# 프로젝트: 반응형 금융 프로필 페이지 구현

## 1. 프로젝트 목표

HTML/CSS 동작 원리 이해 및 금융 상품 비교 애플리케이션 프로필 UI 구현

Bootstrap 5와 커스텀 CSS 통합 시 발생하는 스타일 간섭 및 우선순위 충돌 분석

화면 중단점(768px)에 따른 동적 레이아웃 배치 전환(F305) 완벽 구현

## 2. 주요 트러블슈팅: Bootstrap 스타일 충돌 및 미해결 과제

### 2.1 Bootstrap 도입에 따른 커스텀 CSS 무력화

팀원과의 통합 과정에서 Bootstrap 라이브러리 링크를 추가한 직후, index.html(기본 정보 수정 페이지)에 공들여 작성했던 기존 커스텀 스타일이 완전히 적용되지 않는 심각한 스타일 충돌이 발생했습니다.

발생 현상: 부트스트랩의 기본 스타일 시트가 로드되면서, 기존에 정의한 input-group, 버튼 디자인 등이 라이브러리 기본값으로 덮어씌워짐.

수정 시도 1 (태그 순서 변경): <link> 태그 순서를 조정하여 커스텀 CSS(style_index.css)를 부트스트랩보다 나중에 로드하도록 배치했으나 해결되지 않음.

수정 시도 2 (클래스명 변경): 부트스트랩 내부 클래스와의 이름 충돌을 피하기 위해 input-group을 custom-input-group으로 전면 수정했으나 여전히 스타일이 무시됨.

수정 시도 3 (!important 사용): 명시도를 강제로 높이기 위해 CSS 속성마다 !important를 부여했으나, 부트스트랩의 복합 선택자 위계에 밀려 실패함.

최종 결론: 위와 같은 다각도의 시도에도 불구하고 결국 정확한 원인을 파악하지 못했으며, index.html은 설계한 디자인과 다른 '부트스트랩 기본형' 스타일이 섞인 채로 남아있는 한계가 있음.

## 3. 핵심 기능 구현 성공 사례

### 3.1 네비게이션 및 상단 GNB (성공)

상단바(Top-bar): 명세서의 브랜드 정체성을 반영하여 녹색 계열(#4ca076)의 배경과 흰색 텍스트를 조합한 깔끔한 UI를 구현했습니다.

커스텀 네비게이션: 부트스트랩의 .navbar를 커스터마이징하여 하단 구분선(border-bottom)의 길이를 제한하고 중앙 정렬하는 세밀한 디자인을 완성했습니다.

활성 상태 표시: 현재 머물고 있는 페이지 메뉴에 브랜드 컬러와 굵게 효과를 적용하여 사용자의 위치 인지력을 높였습니다.

### 3.2 반응형 레이아웃 전환(F305) (성공)

명세서 요구사항에 맞춰 768px 중단점을 기준으로 레이아웃이 유동적으로 변하도록 구현 완료했습니다.

데스크톱 모드 (≥768px): portfolio.html에서 자기소개 섹션과 투자 성향 파트가 가로(Row)로 정렬되어 시각적 정보 밀도를 최적화했습니다.

모바일 모드 (<768px): 미디어 쿼리와 부트스트랩의 반응형 유틸리티(flex-column)를 사용하여 투자 성향 파트가 자기소개 밑으로 수직 배치되도록 전환에 성공했습니다.

가변 너비 적용: 모바일 화면에서도 card 내부의 요소들이 부자연스럽게 잘리지 않고 화면 너비에 맞춰 유연하게 리사이징되도록 처리했습니다.

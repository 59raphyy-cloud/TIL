# 관통프로젝트 4회차
## 🎬 AI 기반 영화 정보 서비스
### 1. 프로젝트 개요
- 사용자가 영화 정보(제목, 줄거리, 감독명)를 등록하면, OpenAI API와 Wikipedia API를 활용하여 감독의 상세 프로필과 사진, 대표작을 자동으로 생성하고 관리하는 Django 웹 어플리케이션

### 2-1. 요구사항별 구현 내용
**1. [F401] 영화 정보 등록 (Model & Form)**
- Movie 모델에 기본 정보(제목, 줄거리, 감독명) 외에 AI 생성을 위한 필드(director_info, director_img_url, director_works) 추가
- ModelForm을 활용하여 사용자로부터 필요한 데이터만 입력받도록 구현
- JSONField를 사용하여 감독의 대표 작품 목록(List)을 효율적으로 저장

**2. [F402] 영화 목록 조회 (Index)**
- 전체 영화 목록을 한눈에 볼 수 있도록 메인 페이지 구현
- Bootstrap을 활용하여 깔끔한 리스트 레이아웃 적용

**3. [F403] 영화 상세 정보 조회 (Detail)**
- 사용자가 입력한 정보와 서버에서 생성한 AI 정보를 조합하여 출력
- 부트스트랩 Grid 시스템을 이용해 이미지와 텍스트의 시각적 균형 확보

**4. [F404] 영화 정보 삭제 (Delete)**
- 상세 페이지에서 등록된 영화를 즉시 삭제할 수 있는 기능 구현

### 2-2. 심화 과제
**[F409] AI 기반 감독 정보 자동 생성**
- 데이터 자동화: 사용자가 감독 이름만 입력하면, views.py의 save(commit=False) 단계에서 AI 로직을 호출하여 나머지 필드를 자동으로 채움
- 멀티 API 연동:
    - Wikipedia API: 감독 이름을 기반으로 공식 프로필 이미지를 검색하고 추출
    - OpenAI API (GPT-4o-mini): 감독의 생애, 업적, 대표 작품 3개를 JSON 형식으로 응답받아 가공
- 예외 처리: API 호출 실패 시 "정보 없음" 및 기본 이미지(placehold.jp)를 반환하도록 설계하여 서비스 안정성 확보
- 사용자 안내: 상세 페이지 하단에 AI 생성 정보임을 명시하는 안내 문구 삽입

### 3. 기술 스택 및 라이브러리
- Framework: Django
- AI/API: OpenAI API, Wikipedia API
- Environment: python-dotenv (API Key 보안 관리)
- Frontend: Bootstrap 5 (반응형 UI)

### 4. 학습 기록 및 회고
**HTTP 프로토콜 및 설계 원칙**
- 단순히 GET만 허용하는 것이 아니라, 데이터를 변경하지 않는 안전한 요청의 본질을 준수하기 위해 @require_safe 데코레이터를 사용함. 이를 통해 GET과 HEAD 메서드를 모두 수용하는 표준 규약 준수 설계를 시도함.
- create와 update 과정에서 동일한 URL이 요청 방식(GET: 폼 제공, POST: 데이터 처리)에 따라 어떻게 다른 역할을 수행하는지 이해하고 구조화함.

**Django ORM 및 데이터 처리**
- 사용자 경험을 위해 .order_by('-pk')를 사용하여 최신 등록 영화가 상단에 노출되도록 쿼리셋을 조정함.
- __str__ 메서드를 오버라이딩하여 관리자 페이지 및 쉘 환경에서 데이터 객체가 ID가 아닌 '제목'으로 식별되도록 개선함.
- save(commit=False)를 활용하여 사용자의 입력값이 DB에 최종 반영되기 전, AI 수집 데이터를 주입하는 '데이터 가공 프로세스'를 구축함.

**API 연동 및 통신 전략**
- URL에 데이터를 직접 노출하는 방식 대신 requests의 params 인자를 사용하여 파라미터를 안전하고 구조적으로 전달하는 법을 익힘.
- ai_data.get('info', '기본값') 방식을 사용하여 특정 키가 누락된 불완전한 API 응답 상황에서도 프로그램이 중단되지 않도록 방어적으로 설계함.

**환경 설정 및 트러블슈팅**
- ModuleNotFoundError 발생 시 pip를 통해 openai, python-dotenv, requests 등 필요한 라이브러리를 정확히 식별하고 설치하여 개발 환경을 구축함.
- base.html의 위치를 movies/templates/base.html로 설정하여 여러 앱에서 공통으로 사용할 수 있는 확장성을 고려한 경로 설계를 고민함.

**UI/UX 디테일 개선**
- 영화가 없을 때 나타나는 안내 문구가 줄바꿈되어 레이아웃을 해치는 문제를 해결하기 위해 Bootstrap의 text-nowrap 클래스를 적용하여 시각적 완성도를 높임.
